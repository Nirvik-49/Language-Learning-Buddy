[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Here is our multi-agent architecture diagram. Everything starts with the START node, which passes user input directly into our security_checkpoint node. This node scrubs personal identification details and scans for malicious prompt injections. If the input is flagged, the flow branches immediately to the security_blocked node. If it is safe, it routes to our orchestrator agent. The orchestrator is a coordinator that delegates to three specialized sub-agents: the lesson_planner, the vocabulary_reviewer, and the progress_tracker. Additionally, the orchestrator and sub-agents can consult our local Model Context Protocol server, which exposes tools for fetching daily words, generating quizzes, calculating study streaks, and creating agendas.
